# V1.5 第三批增强 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 补齐变量删除闭环、场景删除后的失效跳转修复，以及分支图问题明细，让 V1.5 的编辑可靠性和诊断能力继续提升。

**Architecture:** 继续沿用现有 feature 内聚 + Zustand persist + graph 纯函数派生模式。第三批只扩展 `editor`、`projects`、`graph` 三条主链，不新增 repository/service 层，不进入拖拽、导出、协作或复杂 DSL。

**Tech Stack:** React 19、TypeScript、Zustand、Vitest、Testing Library

---

## 范围

### 1. 变量删除闭环

- 编辑页变量面板支持删除变量
- 删除变量时清理条件块引用与选项副作用引用
- 删除后修正当前选中变量

### 2. 场景删除后的失效跳转修复

- 删除场景时继续清理 `links`
- 同时把 `choice.metaJson.targetSceneId` 的悬空引用清空

### 3. 分支图问题明细

- 显示问题节点的具体原因
- 覆盖无入边、无出边、无效变量引用、无效跳转目标
- 跟随筛选视图一起收缩

## 范围外

- 不做全局搜索
- 不做复杂结构修复器
- 不做自动批量迁移工具
- 不做拖拽、导出、协作、AI

## 执行顺序

1. 先做变量删除闭环
2. 再做场景删除后的失效跳转修复
3. 最后做图页问题明细

## 验证策略

- 先补失败测试，再补实现
- 先跑 editor / graph / persistence 定向验证
- 最后统一执行：
  - `npm.cmd test`
  - `npm.cmd run build`

## 风险

- 元数据修复逻辑如果写得过粗，容易误伤合法字段
- 图页问题明细若直接写在页面里，会让规则散掉，因此必须继续收敛在 `graphData.ts`
- 历史脏数据需要通过 graph 诊断暴露，但不能因此破坏当前正常编辑流程
