## 当前判断

### 已经完成的 V1.5 目标

- 结构问题提醒：已覆盖无入边、无出边、无效变量引用、无效跳转目标等核心问题
- 角色与场景关联：已支持角色页查看路线与场景关联摘要
- 设定引用关联：已支持设定页查看标题 / 简介 / 正文中的提及场景
- 全局搜索：已支持项目首页按场景 / 角色 / 设定分类搜索
- 路线筛选：已在分支图页落地
- 场景状态管理：已支持 `draft` / `completed` 等状态编辑与展示
- 自动保存与恢复：已稳定可用
- 项目统计：已在项目首页落地

### 仍然缺失或未正式收口的方向

- 项目模板增强尚未进入实现，当前仍没有“结构模板选择与初始化”闭环
- 结构问题提醒仍偏向图结构和失效引用，尚未覆盖“空场景”等更偏创作质量的提醒
- 项目首页仍缺少“最近编辑 / 快速继续创作”一类真正面向日常使用的入口

## 下一阶段目标

### 方向 1：结构治理增强

- 在现有问题明细基础上补齐空场景提醒和问题分类
- 让问题提示更接近“创作风险”而不只是“图结构异常”

### 方向 2：项目模板增强

- 为项目创建补最小模板选择
- 先覆盖线性短篇、多结局、共通线 + 角色线三类高频模板

### 方向 3：项目首页快速继续创作

- 增加最近编辑信息
- 增加继续写作、打开分支图、从头预览等快速入口

## 推荐执行顺序

1. 先做结构治理增强
2. 再做项目首页快速继续创作
3. 最后做项目模板增强

## 原因

- 结构治理增强最直接影响当前已可用主链的稳定性和可读性
- 项目首页快速入口能降低“已经有功能但不够顺”的真实使用阻力
- 模板增强价值明确，但它属于创建期能力，优先级低于当前创作主链的治理收口

## 验证策略

- 每一批先补页面级或纯函数级失败测试，再补实现
- 每一批至少运行对应定向测试
- 每轮收口统一执行：
  - `npm.cmd test`
  - `npm.cmd run build`

## 风险

- 如果结构治理直接写进页面组件，规则会再次分散，后续难以维护
- 如果模板能力过早做深，容易越界到 V2 的“复杂项目脚手架”
- 如果首页快速入口没有和现有 store 真实状态对齐，会出现“入口存在但跳转无效”的伪闭环
