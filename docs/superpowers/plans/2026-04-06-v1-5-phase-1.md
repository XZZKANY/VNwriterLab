# V1.5 第一批增强 Implementation Plan（已完成记录）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不重做架构的前提下，把 V1 从“能用”提升到“顺手”，优先补编辑效率和逻辑表达能力。

**Architecture:** 继续沿用现有 feature 内聚 + Zustand persist + 纯函数预览模式。第一批只扩展 `editor`、`projects`、`preview` 三条主链，不引入新的 repository/service 层，也不进入导出、协作、AI 或复杂图编辑范围。

**Tech Stack:** React 19、TypeScript、Zustand、Vitest、Testing Library

---

## 范围

### 1. 块级删除与排序

- 剧情块支持删除
- 剧情块支持上移 / 下移
- `sortOrder` 在 store 内统一维护

### 2. 路线与场景节点基础操作

- 路线支持最小重命名
- 场景支持同级排序
- 场景支持跨路线移动

### 3. 条件系统升级到条件列表

- 单条件升级为最小条件列表
- 先支持 AND 语义
- 继续复用预览纯函数求值

## 范围外

- 不做拖拽排序
- 不做复杂 DSL 或条件嵌套
- 不做导出、协作、云同步
- 不做复杂图编辑

## 执行顺序

1. 先做块级删除与排序
2. 再做路线与场景节点基础操作
3. 最后做条件列表

## 验证策略

- 每条任务都先写失败测试，再补实现
- 先跑定向测试，再跑全量测试
- 最后统一执行：
  - `npm.cmd test`
  - `npm.cmd run build`

## 风险

- `editor` 与 `project` 双 store 之间仍有结构同步成本，节点移动时需要避免两边状态漂移
- 条件列表如果一次做太深，容易直接越界到 V2，因此第一版只允许 AND 列表
