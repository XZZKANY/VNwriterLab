# 剧情编辑页功能清单

## 一、页面目标

剧情编辑页要解决的不是“排版”，而是：

- 快速写场景
- 插入分支
- 配置条件
- 随时查看上下文
- 不离开当前页面也能继续推进创作

也就是说，这页本质上是一个 **VN 场景搭建器**。

------

# 二、页面结构建议

我建议用 **三栏布局**：

## 左侧：项目结构区

负责“我现在在哪”

## 中间：剧情编辑区

负责“我正在写什么”

## 右侧：辅助信息区

负责“这段内容和谁、什么设定、哪些条件有关”

这个结构很适合 VN 写作，因为作者会不断在：

- 写当前场景
- 查角色设定
- 看前后结构

之间来回切。

------

# 三、左侧：项目结构区

## 1. 项目树

显示层级：

- 路线
- 章节
- 场景

例如：

- 共通线
  - 第1章
    - 场景 1-1
    - 场景 1-2
- 林夏线
  - 场景 A-1
- 会长线
  - 场景 B-1
- 真结局

### 功能

- 展开/收起
- 新建路线
- 新建章节
- 新建场景
- 拖拽排序
- 重命名
- 删除

### 目的

让用户始终知道当前内容在整部作品中的位置。

------

## 2. 当前场景高亮

当前正在编辑的场景要明显高亮。

### 目的

避免用户写着写着搞不清自己在哪个节点。

------

## 3. 快速跳转

支持：

- 上一个场景
- 下一个场景
- 跳到分支目标场景
- 返回上次编辑场景

### 目的

减少来回翻找的成本。

------

## 4. 场景状态标记

每个场景可以显示小状态：

- 草稿
- 已完成
- 待修改
- 缺少跳转
- 有条件
- 结局

### 目的

让项目树不仅是目录，也是进度面板。

------

# 四、中间：剧情编辑区

这是最核心部分。

## 1. 场景标题区

显示：

- 场景名
- 所属路线
- 所属章节
- 场景状态

支持编辑：

- 场景名称
- 场景备注
- 场景标签

------

## 2. 块式编辑器

按块组织内容，而不是一整页纯文本。

建议最少支持 5 类块：

### A. 旁白块

字段：

- 文本内容

作用：

- 场景描写
- 心理活动
- 过渡说明

------

### B. 对白块

字段：

- 角色
- 台词内容
- 语气备注（可选）

作用：

- 写角色发言
- 明确是谁在说话

建议支持：

- 快速选择角色
- 自动带出角色名
- 常用角色快捷输入

------

### C. 选项块

字段：

- 选项文案
- 跳转目标场景
- 选项条件（可选）
- 选项结果（可选）

作用：

- 创建分支点

建议支持：

- 一键新增多个选项
- 每个选项单独设置去向
- 直接从这里新建目标场景

------

### D. 条件块

字段：

- 条件类型
- 条件内容
- 满足时内容 / 不满足时行为

作用：

- 处理分支逻辑
- 控制隐藏剧情
- 控制进入条件

这里一定不要写得太像代码。
 建议界面显示成：

- 当【已见过旧校舍事件】时
- 当【林夏好感 ≥ 3】时
- 否则进入【普通对话】

------

### E. 注释块

字段：

- 作者备注
- 待补内容
- 修改提醒

作用：

- 写作中途记想法
- 标注伏笔
- 留待后续补完

------

## 3. 块操作能力

每个块都应该支持：

- 新增
- 删除
- 复制
- 拖拽排序
- 折叠/展开

### 目的

因为作者经常改顺序、删段落、插分支。

------

## 4. 快速插入入口

在块与块之间显示“+”按钮，快速插入：

- 旁白
- 对白
- 选项
- 条件
- 注释

### 目的

降低操作阻力，让写作节奏不断掉。

------

## 5. 场景内逻辑提示

当用户在本场景里插入了：

- 分支
- 条件
- 标记变更

要在当前场景顶部或侧边显示一个简洁摘要，比如：

- 本场景包含 1 个选项分支
- 修改了 2 个变量
- 依赖 1 个前置条件

### 目的

帮助用户快速理解这个场景的逻辑复杂度。

------

# 五、右侧：辅助信息区

右侧不要做成杂物堆，而要服务当前写作。

建议做成可切换标签页。

## 1. 角色面板

显示当前场景相关角色：

- 角色名
- 简介
- 性格关键词
- 与主角关系
- 当前路线定位

### 作用

写对白时随时核对角色状态。

------

## 2. 设定面板

显示可引用设定：

- 地点
- 术语
- 世界规则
- 相关事件

### 作用

防止设定前后打架。

------

## 3. 条件 / 标记面板

显示：

- 本场景使用了哪些条件
- 本场景改动了哪些标记/变量
- 当前可引用的标记列表

### 作用

方便新手理解“这场戏在逻辑上做了什么”。

------

## 4. 关联场景面板

显示：

- 上游场景
- 下游场景
- 从哪个选项跳来
- 会跳到哪些场景

### 作用

帮助用户在局部写作时不丢上下文。

------

## 5. 注释 / 待办面板

显示：

- 当前场景备注
- 未完成项
- 待补伏笔
- 待检查逻辑

### 作用

让写作过程更连续。

------

# 六、编辑页必须有的交互能力

## 1. 自动保存

这是写作工具必需品。

## 2. 撤销 / 重做

改结构时会频繁用到。

## 3. 快捷键

例如：

- 新建对白块
- 新建旁白块
- 新建选项块
- 保存
- 预览当前场景

## 4. 面板折叠

左侧或右侧要能折叠，让用户专注写内容。

## 5. 从当前场景预览

编辑页里要有一个明显按钮：

**“从这里开始预览”**

这非常关键，因为用户会频繁边写边试。

------

# 七、编辑页里的“低代码”表达方式

这是这页最关键的产品语言问题。

不要优先写成程序术语：

- if
- else
- jump
- flag
- variable

要翻译成创作者语言：

- 满足以下条件时
- 记录这次选择
- 增加角色好感
- 跳转到这个场景
- 未满足时隐藏该选项

这样用户会觉得自己仍然在“写故事”，不是在“学系统”。

------

# 八、剧情编辑页的最小版本

如果你现在只做最小闭环，这页至少要有：

## 左侧

- 项目树
- 当前场景高亮

## 中间

- 场景标题
- 旁白块
- 对白块
- 选项块
- 条件块
- 注释块
- 块排序
- 快速插入

## 右侧

- 角色面板
- 条件面板
- 关联场景面板

## 顶部

- 保存
- 预览当前场景
- 场景状态

这就够支撑第一版核心体验了。

------

# 九、这一页最重要的成功标准

你可以用一句话判断它有没有做好：

**一个不会引擎的人，能不能在这页里顺畅地写完一场带分支的戏。**

更具体一点：

用户能不能在这一页完成：

- 写旁白
- 写对白
- 插入 2 个选项
- 给一个选项加条件
- 设置去向
- 预览结果

如果能，这页就成立了。

------

# 十、以我之见，这一页最该优先打磨的 4 个点

## 1. 对白块输入体验

因为这是最高频操作。

## 2. 选项块配置体验

因为这是 VN 和普通写作工具的分水岭。

## 3. 条件表达方式

因为这是“不会编程的人能不能用”的关键。

## 4. 从当前场景预览

因为这是闭环体验的关键一步。
