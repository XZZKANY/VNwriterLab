# VN 写作工具 PRD（V1 草案）

## 1. 产品名称

暂定：**VN Writer / VN Studio / StoryBranch**

## 2. 产品定位

一款面向**不会引擎的视觉小说创作者**的结构化写作工具，帮助用户把零散的故事灵感整理成**角色、设定、场景、分支、结局**清晰可管理的 VN 结构。

## 3. 目标用户

### 核心用户

有故事想法、想做视觉小说、但不会引擎的人。

### 用户特征

- 会写故事，但不擅长技术实现
- 想做分支剧情、多结局、角色路线
- 目前用 Word / Notion / XMind / Excel 拼凑工作流
- 容易在中长篇创作里把结构写乱

## 4. 核心价值

产品要解决的不是“如何做复杂游戏”，而是：

- **好写**：快速进入创作状态
- **好理清**：看懂剧情结构和分支关系
- **不用编程**：不用学脚本也能写出交互逻辑

可以浓缩成一句：

**写故事，不写代码。**

------

# 5. 用户痛点

## 痛点 1：能写故事，但不会写分支结构

用户能写人物、对白、情绪和设定，但不会把它转成：

- 分支路线
- 条件解锁
- 结局进入逻辑
- 变量影响

## 痛点 2：信息分散

现在常见工作流：

- Word 写正文
- Notion 存设定
- XMind 画分支
- Excel 记条件

导致修改成本高、逻辑容易断。

## 痛点 3：被技术门槛打断

用户不想一开始就接触：

- if / jump / label
- 变量和 flag
- 引擎语法
- 项目结构和导出

------

# 6. V1 产品目标

V1 不追求“完整引擎能力”，只追求一件事：

**让新手作者能把一个 VN 故事从灵感整理成可阅读、可分支、可预览的结构化作品。**

### 成功标准

用户可以在产品内完成：

- 创建一个 VN 项目
- 创建角色和设定
- 写出多个场景
- 插入分支选项
- 配置简单条件
- 通过结构图查看整体路线
- 从头到尾预览一遍故事流程

------

# 7. 功能范围

## V1 必做模块

1. 项目创建与项目结构
2. 剧情编辑器
3. 分支结构视图
4. 角色库
5. 设定库
6. 条件 / 标记系统
7. 简单预览

## V1 不做

- 复杂动画演出
- 高级 UI 编辑
- 美术音频编辑
- 多平台正式导出
- 商业社区
- 团队协作
- 高级脚本系统

------

# 8. 核心用户流程

## 流程 1：开始项目

创建项目
 → 选择模板
 → 输入故事简介
 → 创建主要角色
 → 创建共通线/角色线结构

## 流程 2：写剧情

进入场景
 → 插入旁白/对白
 → 插入选项
 → 配置跳转
 → 配置条件或标记

## 流程 3：整理结构

查看分支图
 → 检查主线/支线
 → 查找断链节点
 → 调整章节顺序和路线结构

## 流程 4：查资料

写作时查看角色卡
 → 查看设定
 → 搜索某个事件/术语
 → 检查角色出场和设定一致性

## 流程 5：预览测试

从起点或某节点开始
 → 点击选项推进
 → 检查路线是否通畅
 → 返回编辑修改

# 10. 模块详细说明

## 模块 A：项目创建与结构管理

### 目标

让用户从“故事”开始，而不是从“技术工程”开始。

### 功能点

- 新建项目
- 输入项目名
- 输入一句话简介
- 选择故事结构模板
- 创建共通线 / 角色线 / 结局目录
- 章节和场景拖拽排序

### 模板建议

- 线性短篇
- 多结局短篇
- 共通线 + 2 条角色线
- 悬疑调查型
- 恋爱路线型

### 价值

减少新手“空白页焦虑”。

------

## 模块 B：剧情编辑器

### 目标

让用户像搭故事一样写 VN，而不是像写代码。

### 编辑块类型

- 旁白块
- 对白块
- 选项块
- 条件块
- 注释块

### 每种块的基础字段

#### 旁白块

- 文本内容

#### 对白块

- 角色
- 文本内容
- 语气备注（可选）

#### 选项块

- 选项文案
- 跳转目标场景
- 影响标记/变量（可选）

#### 条件块

- 显示条件
- 跳转条件
- 条件不满足时行为

#### 注释块

- 作者备注
- 后续待补内容

### 价值

降低写作门槛，统一结构表达。

------

## 模块 C：分支结构视图

### 目标

帮助用户看懂整个作品，而不是只看当前一页内容。

### 功能点

- 用节点表示场景
- 用连线表示跳转
- 区分主线 / 支线 / 结局
- 支持放大缩小和拖动画布
- 点击节点跳回编辑区
- 高亮当前路线
- 标出未连接节点
- 标出无出口节点

### 价值

解决“分支写乱”问题。

------

## 模块 D：角色库

### 目标

让角色信息不再散落在别的文档里。

### 角色字段

- 姓名
- 身份
- 外观
- 性格
- 目标
- 秘密
- 与主角关系
- 所属路线

### 关联信息

- 首次出场场景
- 所有出场场景
- 相关事件
- 相关设定

### 价值

保证人物塑造一致性。

------

## 模块 E：设定库

### 目标

集中管理世界观与规则，减少前后矛盾。

### 分类建议

- 地点
- 世界规则
- 阵营/组织
- 术语
- 重要事件
- 时间线节点

### 功能点

- 新建设定条目
- 添加标签
- 关联角色
- 关联场景
- 搜索术语引用

### 价值

让设定成为可检索、可关联的创作资产。

------

## 模块 F：条件 / 标记系统

### 目标

让用户不用编程也能做“选择影响后续”。

### V1 支持的逻辑类型

#### 布尔标记

- 已见过某角色
- 已触发某事件
- 已进入某路线

#### 数值变量

- 角色好感
- 信任值
- 线索数

### 用户可执行动作

- 选择后记录标记
- 进入场景前检查条件
- 满足条件时显示隐藏选项
- 为角色增加/减少数值

### 界面文案建议

不要显示程序写法，改成：

- “记录这次选择”
- “增加角色好感”
- “满足以下条件时开放”
- “未满足时跳转到”

### 价值

让交互逻辑变成写作者能理解的表达。

------

## 模块 G：简单预览

### 目标

让用户验证剧情是否真的“能走通”。

### 功能点

- 从开头预览
- 从指定场景预览
- 逐步点击选项
- 显示当前角色名和文本
- 显示当前路径信息
- 返回编辑继续修改

### V1 预览范围

- 文本阅读
- 选项分支
- 条件生效
- 简单路线切换

### 不做

- 复杂演出
- 美术音频调度
- 完整游戏 UI

### 价值

快速验证剧情节奏和逻辑。

------

# 11. 页面草图思路

## 页面 1：项目首页

显示：

- 项目名
- 简介
- 主要角色数
- 场景数
- 路线数
- 最近编辑
- 快速进入“继续写作”

## 页面 2：剧情编辑页

布局建议：

- 左侧：项目树（路线 / 章节 / 场景）
- 中间：场景块编辑区
- 右侧：属性面板（角色、条件、注释、关联设定）

## 页面 3：分支图页

布局建议：

- 中间：节点画布
- 左侧：路线筛选
- 右侧：节点详情 / 跳转关系

## 页面 4：角色页

- 左侧角色列表
- 右侧角色卡详情和出场记录

## 页面 5：设定页

- 分类切换
- 设定条目列表
- 条目详情和关联内容

## 页面 6：预览页

- 对话框阅读区域
- 选项按钮
- 当前场景信息
- 返回编辑按钮

------

# 12. MVP 验收标准

### 创作层

用户能完成：

- 1 个项目
- 3 个角色
- 5 个以上场景
- 1 个分支点
- 1 个结局
- 1 个条件判断

### 产品层

用户能明确感受到：

- 比普通文档工具更适合写 VN
- 比脑内管理更清楚
- 比直接上引擎更容易开始
