# V1 页面清单

我建议 V1 先控制在 **6 个核心页面**，不要太散。

## 1. 项目首页

作用：

- 进入项目
- 看项目概况
- 快速继续创作

## 2. 剧情编辑页

作用：

- 写场景
- 插入对白、选项、条件
- 从当前场景预览

## 3. 分支图页

作用：

- 看整体结构
- 找问题
- 调整路线关系

## 4. 角色页

作用：

- 管理角色资料
- 查看角色关联场景

## 5. 设定页

作用：

- 管理地点、术语、规则、事件

## 6. 预览页

作用：

- 阅读和测试当前作品流程

------

# 一、项目首页

## 页面目标

让用户一进项目就知道：

- 这个项目是什么
- 现在进度到哪
- 下一步该点哪里

## 核心信息

- 项目名
- 一句话简介
- 最近编辑时间
- 路线数
- 场景数
- 角色数
- 分支数
- 结局数

## 核心模块

### 1. 项目概览卡

显示：

- 项目名称
- 类型
- 简介
- 当前状态

### 2. 最近编辑

显示最近编辑过的：

- 场景
- 角色
- 设定

### 3. 创作进度概览

显示：

- 已完成场景
- 草稿场景
- 未连接节点
- 待补内容

### 4. 快速入口

快速跳到：

- 继续写上次场景
- 新建场景
- 打开分支图
- 从开头预览

## 核心按钮

- **继续写作**
- **新建场景**
- **打开分支图**
- **从头预览**
- **新增角色**
- **新增设定**

------

# 八、V1 真正必须做的页面优先级

如果资源很紧，我会这样排：

## 第一优先级

1. 剧情编辑页
2. 分支图页
3. 预览页

这三个决定产品是不是成立。

## 第二优先级

1. 项目首页
2. 角色页
3. 设定页

这三个决定产品是不是完整、顺手。

# 九、最小上线版本可以再砍一刀

如果你要做更小的 MVP，甚至可以先只上线：

- 项目首页
- 剧情编辑页
- 分支图页
- 预览页

然后把角色、设定先做成编辑页右侧的简化面板，而不是独立页面。

这是更省资源的做法。

------

# 十、我对 V1 页面设计的判断

以我之见，**V1 最重要的是保证“编辑页 ↔ 分支图页 ↔ 预览页”三者切换非常顺**。

因为用户的真实操作会一直循环：

**写一段 → 看结构 → 试一遍 → 回来改**

这个闭环顺了，产品就会有生命力。
